LoRa 長距離無線通訊技術之應用:以 EcoScout環境數據採集器之實作為例

簡介

  在這個由數據組成的時代，所有的分析都需要大量數據的收集，但在一些極端氣候或者危險環境的地方，例如車諾普、張家界……等，假設人類要長期在當地架設檢測站收集數據，不僅耗時耗力，還很危險，更有可能因為交通不便造成物資不便傳輸或使其與外界失聯，而這個時候如果有一個機器可以替你收集環境資料並將期都回傳到主機，那一定會使人類更有效的了解不同環境的生態。
  EcoScout 上部署了很多傳感器和任務，包括溫濕度傳感器、亮度傳感器、土壤傳感器等。 除了部署很多傳感器感知環境之外，並使用 LoRA 作為傳輸的媒介，並外加電源使其在野外可以無網路的情況下傳輸。
  在野外或危險環境中收集環境數據必須面臨一個非常重要的問題，那就是能源消耗。因此，我們使用 meCreate 作為開發平台。除了更高效的部署之外，meCreate 還可以根據能源關鍵程度將任務分成多個版本，並在必要時切換到省電版本。高能量關鍵版本中任務的 WCET 低於低能量關鍵任務版本，因此 CPU 可以獲得更多空閒時間。當 CPU 獲得更多空閒時間時，meCreate 會降低 CPU 頻率以節省電量。在這個過程中，meCreate 仍然會確保工作不會錯過截止日期。

開發環境

客戶端:採用樹梅派加上多個傳感器，外加上一個LoRA，一個負責發送訊息到伺服器端

伺服器端:採用樹梅派架設LAMP伺服器負責接收訊息

在專題的製作過程中，多多少少都會遭遇到一些問題，不管事技術層面，或者是應用上。以下列出幾點問題：

  1.對於樹梅派開發環境的不熟悉

    網路資訊查詢或尋求學長協助

  2.硬體設備上的需求

    尋求指導老師幫忙或善用學校原有設備

  3.對於 UART、SPI、I2C 等接口連接

    在系統檔內修改或網路資訊查詢

  4.戶外的供電方式

    多個電池並聯

  5.點對點的資料傳輸

    使用 LORA 這種低功耗、遠距離的傳輸方式，使主機可以獲取資訊。

  6.Pi4 容易熱當

    加入風扇

  7.如何展示結果

    利用 LAMP 架設網站
